## 성능 테스트 시작 전 알아야 할 사전 지식 정리

### 지연시간과 처리량

+ 지연시간 (Latency)
    - 요청을 보내고 응답을 받기까지 걸리는 시간
    ```java
    long startTime = System.currentTimeMillis();
    String result = userService.getUserInfo(userId); // API 호출
    long endTime = System.currentTimeMillis();

    long latency = endTime - startTime; // 이게 지연시간 (예: 50ms)
    ```
    - 사용자가 직접 체감하는 성능 지표
    - '로딩이 느려요' => 보통 지연시간이 길다
+ 처리량 (Throughput)
    - 단위 시간당 처리할 수 있는 요청의 수
        + TPS로 측정 (Transactions Per Second)
        + 동시에 많은 트래픽을 소화가능하다 = 처리량이 높다
+ 지연시간과 처리량의 관계
    - 지연시간은 사용자의 입장, 처리량은 시스템의 입장
    - 지연시간은 개인의 속도
        ```
        "내가 얼마나 빨리 응답받을 수 있나?"

        API 실행 -> 응답 100ms
        ```
    - 처리량은 시스템의 용량
        ```
        "시스템이 동시에 몇명을 받을 수 있나?"

        1초에 100명 처리 가능
        ```
    - 시나리오
        ```java
        @GetMapping("/ratings/{id}")
        public Rating getRating(@PathVariable Long id) {
            // DB 조회
            return ratingRepository.findById(id);  // 100ms
        }
        ```
        + 상황 1 : 사용자 1명
            - 요청 100ms 후 응답
            - 지연시간 : 100ms
            - 처리량 : 10TPS (1초 / 0.1초 = 10)
        + 상황 2 : 사용자 10명
            ```
            // 순차 처리 (Single Thread)
            10:00:00.000 - 사용자 A 요청 시작
            10:00:00.100 - 사용자 A 응답 완료
            10:00:00.100 - 사용자 B 요청 시작
            10:00:00.200 - 사용자 B 응답 완료
            ...
            10:00:01.000 - 사용자 J 응답 완료
            ```
            - 각 사용자의 지연시간 100ms씩 증가
            - 지연시간 : 평균 550ms
            - 처리량 : 10TPS
        + 여기서 지연시간을 줄이면?
            - 처리속도가 빨라짐
            - 같은 시간에 더 많이 처리 가능
            - 처리량 증가
        + 처리량을 늘리면?
            - 싱글 스레드를 멀티 스레드 환경으로 변경
            - 지연시간은 여전히 100ms지만 처리량은 100TPS로 증가
    - 정리
        + 지연시간
            - 개인이 느끼는 속도
            - "내 요청이 얼마나 빠르게 처리되나?"
            - 로딩시간
        + 처리량
            - 시스템이 감당가능한 용량
            - "동시에 몇명까지 받을 수 있나?"
            - 동시 처리 능력
        + 관계
            - 지연시간 개선 -> 처리량도 개선
            - 처리량 개선 -> 지연시간 유지 (동시 처리만 증가)
