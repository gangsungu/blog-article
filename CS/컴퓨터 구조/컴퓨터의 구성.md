## 컴퓨터의 구성

### 컴퓨터의 구성 요소
+ 하드웨어
    - 컴퓨터의 물리적인 장치
    - 중앙 처리 장치 (CPU)
        + 컴퓨터의 두뇌, 연산과 제어 담당
        + 명령어를 처리하고 실행 순서를 제어
    - 기억 장치 (메모리)
        + 컴퓨터의 기억 공간
        + 주기억 장치 (RAM) : 실행 중인 프로그램과 데이터를 임시 저장
        + 보조기억 장치 (SSD, HDD) : 영구 저장
    - 입출력 장치 (I/O 장치)
        + 사람과 컴퓨터 간의 정보 교환
        + 입력장치 : 키보드, 마우스 등
        + 출력장치 : 모니터, 스피커, 프린터 등
+ 소프트웨어
    - 하드웨어의 동작을 제어하는 프로그램
    - 응용 소프트웨어
        + 사용자가 목적을 위해 실행하는 프로그램
        + 게임, 웹 브라우저 등
    - 시스템 소프트웨어
        + 하드웨어와 응용 소프트웨어가 원활히 동작하도록 지원
        + 운영체제, 컴파일러, 드라이버 등

### 시스템 버스
+ 시스템 버스
    - 하드웨어 구성 요소를 물리적으로 연결하는 선
    - 데이터 버스
        + 실제 데이터를 전송하는 통로
        + CPU와 메모리, CPU와 I/O 장치 사이에서 데이터를 주고받는 통로 (양방향)
        + 버스의 폭(width)가 성능을 결정 (8bit, 16bit 등)
        + CPU가 메모리에서 숫자 31을 읽어올 때 사용
    - 주소 버스 (단방향)
        + 어디에 있는 데이터를 읽거나 쓸지 위치를 지정
        + CPU에서만 출발 (단방향)
        + 버스의 폭이 접근 가능한 메모리 크기를 결정
        + 32bit 주소 버스 : 2<sup>32</sup> = 4GB의 메모리 주소 지정 가능
    - 제어 버스
        + 데이터 전송의 동작상태를 알리는 제어 신호 통로
        + 읽기/쓰기 신호, 인터럽트, 클록 등
    - 왜 중요한가
        + 버스의 속도가 느리면 아무리 CPU가 빨라도 전체 성능이 떨어진다 (병목현상)
        + 버스 규격에 따라 연결할 수 있는 장치와 메모리 크기가 결정 (확장성)
    - 실제 동작 예시
        ```
        1. 주소 버스: CPU가 "0x1000" 주소를 전송
        2. 제어 버스: "읽기(READ)" 신호 전송
        3. 데이터 버스: 메모리가 해당 주소의 데이터를 CPU로 전송
        ```
